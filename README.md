# 软件可靠性分析与预测平台

本平台是一个集成多种经典软件可靠性增长模型（SRGM）的 Web 分析系统。它旨在帮助开发人员和质量保证工程师通过历史失效数据，评估软件当前的可靠性水平，并预测未来的失效趋势。

## 🌟 核心功能

- **用户认证与管理**：
  - **用户注册**：支持新用户创建账号。
  - **用户登录/登出**：安全的会话管理，确保数据隐私。
- **多模型支持**：
  - **JM 模型 (Jelinski-Moranda)**：经典的二项分布模型，适用于失效间隔时间分析。
  - **GO 模型 (Goel-Okumoto)**：基于非均匀泊松过程（NHPP）的指数增长模型。
  - **NHPP 模型**：支持多种增长曲线（如指数型、S型等）的非均匀泊松过程模型。
  - **BP 神经网络**：利用深度学习（人工神经网络）对复杂的非线性失效序列进行时间序列预测。
  - **ARIMA 模型**：时间序列分析模型，用于预测未来的失效趋势。
  - **GM(1,1) 模型**：灰色预测模型，适用于小样本数据预测。
  - **SVR 模型**：支持向量回归模型，用于非线性回归预测。
- **数据管理**：
  - **文件上传**：支持 CSV/TXT 格式的失效数据文件上传。
  - **数据集列表**：用户可查看和管理自己上传的所有数据集。
  - **数据删除**：支持删除已上传的数据集及其物理文件。
  - **在线预览**：支持在模型页面选择已上传数据集进行分析。
- **可视化分析**：自动生成失效预测曲线、可靠度趋势图以及模型训练损失图。
- **精度评估**：提供 MAE、MSE、RMSE、R² Score 等多种指标，量化预测结果的准确性。

## 🛠️ 技术栈

- **后端**：Python 3.8+, Flask, Flask-SQLAlchemy
- **算法库**：NumPy, Scikit-learn, SciPy, Statsmodels
- **前端**：HTML5, CSS3, JavaScript (原生 JS + Chart.js/ECharts 用于绘图)
- **数据库**：SQLite (默认，无需额外配置即可运行，数据文件位于 `instance/reliability_data.db`)

## 🚀 快速开始

### 1. 环境准备

建议使用 Python 3.8 及以上版本。建议创建**虚拟环境**：

```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
.\venv\Scripts\activate  # Windows
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动应用

在项目根目录下执行：

```bash
python app.py
```

应用将默认运行在：`http://127.0.0.1:5000`

## 📊 模型使用指南

### 数据输入格式要求

- **格式**：逗号分隔的数值序列或每行一个数值（支持 `.csv` 或 `.txt` 文件）。
- **物理含义**：**失效间隔时间 (Inter-failure times)**。例如 `[9, 12, 11]` 表示第1次失效发生在第9小时，第2次与第1次间隔12小时，依此类推。
- **注意**：部分模型对数据量有最小要求（通常建议至少 5-10 个数据点）。

### 训练与测试划分

系统支持在页面上通过滑动条调整**训练比例**（默认为 70%）。

- **训练集**：用于估算模型参数或训练神经网络权重。
- **测试集**：用于验证模型的预测精度。

## ⚠️ 注意事项与常见问题

1. **模型收敛性**：
   - **JM/GO/NHPP 模型**：由于使用数值优化算法（如极大似然估计），如果输入数据极不符合模型假设，可能会出现参数无法收敛的情况。
   - **BP/ARIMA/GM(1,1)/SVR 模型**：BP 神经网络作为随机梯度下降算法，多次训练的结果可能略有差异。建议根据数据量调整“滑动窗口（Look-back）”和“学习率”。其他模型也可能因数据特性导致预测效果不佳。
2. **上传文件**：请确保上传的 CSV/TXT 文件编码为 `UTF-8`，且内容只包含数值和逗号/换行符。
   - 示例1：
     ```
     1
     2
     3
     4
     5
     ....
     ```
   - 示例2：
     ```
     1,2,3,4,5,....
     ```
3. **性能**：部分复杂模型（如 BP 神经网络）在高轮数（Epochs）训练时会消耗一定的 CPU 资源，计算期间页面可能会有短暂延迟。

## 📁 项目结构

- `app.py`: Web 服务主程序，包含路由逻辑、API 定义和数据库配置。
- `model/`: 可靠性算法核心实现，包含多种模型的预测逻辑。
- `templates/`: HTML 模板文件，定义了用户界面的结构。
- `static/`: CSS 样式及静态资源（如 JavaScript 文件）。
- `uploads/`: 用户上传的数据集物理文件存储目录。
- `instance/reliability_data.db`: 应用程序的 SQLite 数据库文件，包含用户和数据集信息。
